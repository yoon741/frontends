폼 관련 태그

개요

html에서 사용자의 정보를 입력받아 서버로 전송하려면 form 요소를 사용해야 함

form 태그를 사용해서 입력받은 데이터를 서버로 어떻게 보낼것인지 (method), 전송할 데이터의 유형은 무엇인지 (enctype), 전송된 데이터는 서버에서 누가 처리할 지 (action)등을 지정해야 함

입력받은 데이터 서버로 보내기

  • get방식은 url에 질의문자열querystring 형태로 데이터를 작성해서 전송함. 따라서, 입력한 데이터가 그대로 노출. 또한, 전송할 수 있는 데이터의 크기가 제한적임. 중요도가 낮은 정보를 입력 폼없이 임의로 전달시 주로 사용
  • post방식은 데이터를 http header에 별도로 작성해서 전송함. 따라서, 데이터가 외부로 보이지 않으며, 전송할 수 있는 데이터의 크기가 무제한. 또한, 텍스트와 바이너리를 같이 보낼수 있어서 보안성과 활용성이 좋음

질의문자열querystring

클라이언트가 서버로 정보를 요청시 정보전달 양식에 의해 작성된 문자열을 의미.

질의문자열의 시작은 ?로, 전달하려는 데이터는 키와 값의 형태로 작성하며, 각각의 정보는 &로 연결함

영문자,숫자,일부특수문자는 그대로 전송되지만, 그외 나머지는 %와 16진수코드로 인코딩된 문자로 변환되어 전송됨 (예: 수지는 %EC%88%98%EC%A7%80로 인코딩됨 - urlencoder.org)

다양한 input 태그

html5에서는 다양한 input요소 태그를 사용해서 사용자로부터 데이터를 입력받을 수 있음

  • text : 한줄 텍스트를 입력받음
  • password : 한줄 비밀 텍스트를 입력받음, 입력시 별표문자가 화면에 표시
  • radio : 여러 선택사항 중 하나만 입력받음
  • checkbox : 여러 선택사항 중 다수를 입력받음
  • select : 여러 선택사항 중 하나만 입력받음, 단, 선택할 수 있는 사항은 드롭다운 메뉴로 보여짐
  • textarea : 여러 줄 텍스트를 입력받음
  • button : 사용자가 클릭할 수 있는 버튼을 표시, type속성으로 버튼의 종류를 지정함. button은 일반적인 버튼을 표시, 보통 자바스크립트와 함께 사용 submit은 입력한 데이터들을 서버로 전송시 사용 reset은 입력한 데이터들을 모두 지우고 초기화할 때 사용
  • file : 텍스트가 아닌 파일을 서버로 전송할때 사용, 단, enctype이 multipart/form-data로 설정되어야 함 또한, 서버측에서는 텍스트만 처리하는 방식이 아닌 다른 방식으로 처리해야 함
  • label : 폼 요소에 제목을 작성할때 사용
  • number: 숫자만 입력가능, 상하 화살표를 눌러 숫자를 1씩 증가시키거나 감소시킬수 있고, 직접 입력도 가능
  • tel: 전화번호 입력 구현시 사용, 스마트폰이나 일부 디바이스에서 tel속성이 지정된 input 필드를 만나면 쿼티 자판 대신 전화번호를 입력할 수 있는 다이얼패드를 띄움
  • search: 문잔나 숫자를 입력하게 되면 입력창 우측에 x표시가 나타남. 클릭하면 입력창에 입력한 내용이 삭제됨
  • email: 이메일 형식으로 입력이 필요할 경우 사용하면, 입력한 내용이 이메일 형식인 @과 url형식이 맞는지 어느정도 판별가능 option value를 사용하면 @뒤 url형식을 지정할 수 있다
  • url: 올바른 형태로 쓰지 않으면 새로 입력하라는 메세지가 나옴
  • color:
  • range:
  • date:
  • time
  • datalist
  • meter
  • progress

공통 input 속성

value : 폼 요소에 기본값 지정

readonly : 폼 요소에 입력한 값을 수정하지 못하게 막음

disabled : 폼 요소가 서버로 전송되지 못하게 막음

placeholder : 폼 요소내에 레이블을 표시할 수 있게 해 줌

size, maxlength : 폼 요소의 너비와 입력가능 최대 글자수를 지정

required : 폼 요소에 값을 반드시 입력해야 하는지 여부 지정
